     Plan: RF-DETR Fine-Tuning Codebase

     Context

     The user has a minimal Kaggle notebook (rf-detr.ipynb) that fine-tunes
     RF-DETR models on the obstacle-detection dataset (25 classes, 24,326
     images, YOLO format). The goal is to convert this into a proper CLI
     codebase that can be cloned from GitHub and run in Kaggle with !pip install
     -r requirements.txt then !python main.py.

     Dataset source: /kaggle/input/datasets/abtinzandi/obstacle-detection-datase
     t/ROD-Dataset/dataset
     Processed output: /kaggle/working/dataset_{resolution} (e.g., dataset_384)
     Key insight from rfdetr library research:
     - Checkpoint file is checkpoint_best_total.pth, not best.ckpt
     - model.evaluate() does NOT exist — run manual eval with model.predict() +
     supervision
     - model.predict() returns sv.Detections directly (no .from_inference()
     needed)
     - TensorBoard + CSV logging is built into rfdetr's PTL trainer
     automatically
     - Do NOT pass callbacks={} — it is deprecated/discarded in current versions
     - rfdetr uses subdirs train/, valid/, test/ (ignores YAML path entries)

     class TeeLogger:
         def __init__(self, filepath):

             self.log = open(filepath, "a", buffering=1)
         def write(self, message):
             self.terminal.write(message); self.log.write(message)
